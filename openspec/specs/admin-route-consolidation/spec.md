## ADDED Requirements

### Requirement: Generic CRUD route factory exists

A route factory module at `src/routes/admin/routeFactory.js` SHALL provide a `createCrudRoutes()` function that generates standard Express CRUD routes from a configuration object.

#### Scenario: Factory generates list route
- **WHEN** a CRUD configuration specifies a `listMethod`
- **THEN** the factory SHALL generate a `GET /` route that calls the configured list method, applies pagination from query parameters, and returns a standardized JSON response

#### Scenario: Factory generates get-by-id route
- **WHEN** a CRUD configuration specifies a `getMethod`
- **THEN** the factory SHALL generate a `GET /:id` route that calls the configured get method with the ID parameter and returns a standardized JSON response

#### Scenario: Factory generates create route
- **WHEN** a CRUD configuration specifies a `createMethod`
- **THEN** the factory SHALL generate a `POST /` route that validates required fields from the request body, calls the configured create method, and returns a standardized JSON response

#### Scenario: Factory generates update route
- **WHEN** a CRUD configuration specifies an `updateMethod`
- **THEN** the factory SHALL generate a `PUT /:id` route that validates input, calls the configured update method, and returns a standardized JSON response

#### Scenario: Factory generates delete route
- **WHEN** a CRUD configuration specifies a `deleteMethod`
- **THEN** the factory SHALL generate a `DELETE /:id` route that calls the configured delete method and returns a standardized JSON response

### Requirement: Factory applies standard middleware

All routes generated by the factory SHALL include admin authentication middleware and standardized error handling.

#### Scenario: Auth middleware is applied
- **WHEN** any factory-generated route is accessed
- **THEN** the `authenticateAdmin` middleware SHALL be applied before the route handler executes

#### Scenario: Errors are caught and formatted
- **WHEN** a factory-generated route handler throws an error
- **THEN** the error SHALL be caught and returned as a standardized JSON error response with appropriate HTTP status code

### Requirement: Account admin routes use the factory

The 12 account-type admin route files SHALL be migrated to use `createCrudRoutes()`. Routes with non-CRUD behavior (dashboard, sync, system, concurrency, balance scripts) SHALL keep their custom implementations.

#### Scenario: Claude accounts use factory
- **WHEN** the Claude accounts admin route file is examined
- **THEN** it SHALL use `createCrudRoutes()` and SHALL be no more than 40 lines

#### Scenario: Non-CRUD routes stay custom
- **WHEN** the dashboard or system admin route files are examined
- **THEN** they SHALL retain their custom route handlers without using the factory
